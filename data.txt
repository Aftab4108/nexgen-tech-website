# Automated Website Deployment Using Docker & GitHub

I developed a responsive static company website using HTML, CSS, and JavaScript to simulate a real-world business website. .
The project was designed to demonstrate fundamental DevOps concepts, including version control, containerization, deployment, 
and application accessibility.

To ensure consistency across different environments, I containerized the website using Docker and served it through an Nginx 
web server running inside a Docker container. This approach allowed the application to run reliably regardless of the underlying 
operating system or machine configuration.

I used Git for source code management and GitHub as the central repository for storing, tracking, and maintaining project versions. 
This enabled efficient version control, change tracking, and collaboration practices commonly used in software development teams.

As part of the deployment workflow, I created a Dockerfile to build a Docker image from the website source code and deployed the
application as a Docker container. This simplified the deployment process and ensured a consistent runtime environment.

To make the locally hosted application accessible over the internet, I used ngrok to create a secure public tunnel. This allowed 
external users to access and test the website through a public URL without requiring deployment to a cloud platform.

Through this project, I gained hands-on experience with Docker image creation, container management, Git and GitHub workflows, Nginx 
web server configuration, and deployment processes. The project strengthened my understanding of DevOps practices, including application 
packaging, environment consistency, source code management, and basic CI/CD concepts.

**how to run**
# open Docker Desktop first and wait until its run

# ========================================================================================
# COMPANY WEBSITE DOCKER DEPLOYMENT COMMANDS
# ============================================

# 1. Build Docker image
docker build -t company-website 

# 2. Remove any old container with same name (ignore if none)
docker rm -f my-site 2>$null

# 3. Run the container (maps port 8080 on your PC to port 80 inside container)
docker run -d -p 8080:80 --name my-site company-website

# 4. Open website in default browser
Start-Process "http://localhost:8080"

# 5. (Optional) Share website publicly using ngrok (install ngrok first)
#    Uncomment the line below after installing ngrok and setting authtoken
#    If web site link is not opning run this command on command prompt 
# ngrok http 8080

# ========================================================================================
# USEFUL COMMANDS
# ============================================

# Stop the container
# docker stop my-site

# Start the container again
# docker start my-site

# See running containers
# docker ps

# View container logs
# docker logs my-site

# Remove container and image (when done)
# docker rm -f my-site
# docker rmi company-website
